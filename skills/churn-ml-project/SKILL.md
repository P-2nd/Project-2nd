# 고객 이탈 예측 ML 프로젝트

작고 재현 가능한 모델 비교 프로젝트를 유지한다. 별도 요청이 없으면 서비스, 대시보드, 배포 코드는 범위에서 제외한다.

## 계층별 작업 위치

- 원본 데이터는 `data/raw/`, 변환 데이터셋은 `data/processed/`, 지표·예측값·그래프는 `data/evaluation/`에 저장한다.
- 재사용 코드는 `src/`에 둔다. 전처리, 학습기, 평가, 공통 유틸리티 사이에 중복 코드를 만들지 않는다.
- 모델 하이퍼파라미터와 실험 공통 상수는 `configs/`에서 관리한다.
- 직렬화된 모델과 학습된 전처리기는 `models/`에 저장한다. 비밀 정보와 원본 개인정보는 커밋하지 않는다.
- 테스트 코드는 `tests/`, 탐색용 노트북은 `notebooks/`에 둔다.

## 실험 타당성 유지

1. 결측치 처리기, 인코더, 스케일러, 피처 선택기, 샘플링 기법은 데이터를 분할한 뒤 학습한다.
2. 이진 이탈 라벨에는 가능한 경우 층화 분할을 적용하고, 난수 시드를 기록한다.
3. 100%, 50% 실험에서는 동일한 테스트 세트를 고정하고 학습 데이터만 축소한다.
4. 모든 모델을 Accuracy, Precision, Recall, F1, ROC-AUC, 혼동 행렬, 추론 지연 시간으로 평가한다.
5. 실행별 설정, 지표, 임계값, 아티팩트 경로를 `data/evaluation/`에 저장한다.

## 문서 정합성 유지

- `EDA.md`는 빈 템플릿이 아니라 측정된 데이터 사실을 기준으로 갱신한다.
- `RESULT.md`는 저장된 평가 결과를 기준으로 갱신한다.
- `README.md`, `RESULT.md`, 설정 파일, 소스 코드의 모델 목록을 일치시킨다.
